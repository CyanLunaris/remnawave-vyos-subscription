# Remnawave Sync — Design Document

**Date:** 2026-03-05
**Status:** Approved

---

## Overview

Service for VyOS that synchronises VPN node configs from a Remnawave panel subscription URL, runs sing-box as a transparent TUN proxy, and automatically rotates nodes on failure.

---

## Architecture

```
Remnawave Panel
    ↓  GET /sub/{token}
remnawave-sync.py  →  nodes.json (cache)
    ↓
config generator  →  /etc/sing-box/config.json
    ↓
sing-box (tun0)   →  Internet (via current node)
    ↑
remnawave-heartbeat.py  →  auto-rotate on failure
```

---

## File Layout

```
/etc/remnawave/
  config.env          # runtime configuration
  nodes.json          # parsed node list (cache)
  state.json          # {current_node: 0, fail_count: 0, last_switch: ""}

/usr/local/bin/
  sing-box            # downloaded automatically
  remnawave-sync.py   # subscription sync + config generation
  remnawave-heartbeat.py  # connectivity check + node rotation

/etc/sing-box/
  config.json         # generated by sync script
  geoip.db            # downloaded automatically
  geosite.db          # downloaded automatically

/var/log/remnawave/
  sync.log
  heartbeat.log

/etc/systemd/system/
  sing-box.service
  remnawave-sync.service
  remnawave-sync.timer
  remnawave-heartbeat.service
  remnawave-heartbeat.timer
```

## Configuration (`/etc/remnawave/config.env`)

```bash
SUBSCRIPTION_URL=https://panel.example.com/sub/TOKEN
SYNC_INTERVAL=10min
HEARTBEAT_INTERVAL=30s
HEARTBEAT_HOST=cp.cloudflare.com
HEARTBEAT_FAIL_THRESHOLD=2      # consecutive failures before switching
HEARTBEAT_TIMEOUT=5             # seconds

GEO_DIRECT_IP=private,ru        # comma-separated geoip codes routed directly
GEO_DIRECT_SITE=ru              # comma-separated geosite codes routed directly
TUN_INTERFACE=tun0
TUN_ADDRESS=172.19.0.1/30
```

## remnawave-sync.py

1. Download/verify sing-box binary (GitHub releases)
2. Download/verify geoip.db + geosite.db
3. GET `$SUBSCRIPTION_URL`
4. Detect response format:
   - Base64-encoded → decode
   - Plain text with `vless://` / `vmess://` / `trojan://` lines
5. Parse each URI into structured node dict
6. Write all nodes to `/etc/remnawave/nodes.json`
7. Load `state.json` → get current node index
8. Generate `/etc/sing-box/config.json`
9. Compare hash with previous config → if changed, reload sing-box

---

## URI Parsing

| Scheme | Format |
|--------|--------|
| `vless://` | `vless://uuid@host:port?security=reality&flow=...&pbk=...&sni=...` |
| `vmess://` | `vmess://base64(json)` |
| `trojan://` | `trojan://password@host:port?sni=...` |

All other schemes are skipped with a warning.

## sing-box Config (Generated)

```json
{
  "inbounds": [{
    "type": "tun",
    "interface_name": "tun0",
    "inet4_address": "172.19.0.1/30",
    "auto_route": false,
    "stack": "system"
  }],
  "outbounds": [
    { "tag": "proxy", "type": "<protocol>", ... },
    { "tag": "direct", "type": "direct" }
  ],
  "route": {
    "rules": [
      {"ip_is_private": true, "outbound": "direct"},
      {"geoip": ["ru"], "outbound": "direct"},
      {"geosite": ["ru"], "outbound": "direct"}
    ],
    "final": "proxy",
    "geoip": {"path": "/etc/sing-box/geoip.db"},
    "geosite": {"path": "/etc/sing-box/geosite.db"}
  }
}
```

## remnawave-heartbeat.py

```
1. Load state.json + nodes.json
2. HTTP GET https://{HEARTBEAT_HOST} with timeout
3. On success → reset fail_count to 0
4. On failure → increment fail_count
5. If fail_count >= HEARTBEAT_FAIL_THRESHOLD:
   - Rotate: current_node = (current_node + 1) % len(nodes)
   - Reset fail_count = 0
   - Regenerate config.json
   - systemctl reload sing-box
   - Log rotation event
6. Update state.json
```

## Systemd Units

- `sing-box.service` — Type=simple, Restart=always
- `remnawave-sync.service` — Type=oneshot
- `remnawave-sync.timer` — OnBootSec=10s, OnUnitActiveSec=10min
- `remnawave-heartbeat.service` — Type=oneshot
- `remnawave-heartbeat.timer` — OnBootSec=30s, OnUnitActiveSec=30s

## VyOS Integration

After TUN interface is up, add routing rules via VyOS config:

```
set interfaces tunnel tun0 ...   # or via ip route on VyOS
set policy route ... (PBR for LAN → tun0)
```

An install script will handle initial setup of:
1. ip tuntap add dev tun0 mode tun
2. ip addr add 172.19.0.1/30 dev tun0
3. ip route add default dev tun0 (for proxy traffic)

---

## Error Handling

- Subscription fetch fail → keep existing nodes.json, log error
- All nodes exhausted → cycle back to first node, alert in log
- sing-box fails to start → log error, do not crash heartbeat
- Malformed URI → skip with warning, continue parsing rest
